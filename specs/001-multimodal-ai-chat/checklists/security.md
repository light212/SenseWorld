# Security & Privacy Checklist: 多模态 AI 对话平台

**Purpose**: 确保实现满足安全与隐私合规要求  
**Created**: 2026-03-25  
**Feature**: [spec.md](../spec.md)

## 身份认证与授权

- [x] 使用 JWT 进行无状态认证
- [x] 密码使用 bcrypt 哈希存储
- [x] Token 过期时间合理设置（24小时）
- [x] WebSocket 连接需要有效 Token
- [ ] 实现 Token 刷新机制（待后续迭代）
- [ ] 实现登录失败次数限制（待后续迭代）

## 数据传输安全

- [x] 所有 API 支持 HTTPS
- [x] WebSocket 使用 WSS 加密
- [x] CORS 配置仅允许指定域名
- [x] 敏感数据不在 URL 参数中传输

## 数据存储安全

- [x] 用户密码仅存储哈希值
- [x] 数据库连接使用参数化查询（SQLAlchemy ORM）
- [x] 敏感配置通过环境变量管理
- [x] Redis 缓存数据设置 TTL
- [ ] 实现数据加密存储（待评估需求）

## 音频数据处理

- [x] 音频数据实时处理，不持久化存储原始音频
- [x] ASR 转写结果仅保存文本，不保存音频
- [x] 文件上传大小限制（25MB）
- [x] 文件类型验证（仅允许音频格式）

## API 安全

- [x] 所有敏感端点需要认证
- [x] 输入验证使用 Pydantic 模型
- [x] 全局异常处理避免敏感信息泄露
- [x] 错误响应不暴露内部实现细节
- [ ] 实现请求频率限制（待后续迭代）
- [ ] 实现 API 密钥轮换机制（待后续迭代）

## 第三方服务安全

- [x] OpenAI API Key 通过环境变量管理
- [x] API Key 不在日志中打印
- [ ] 实现 API Key 使用监控（待后续迭代）

## 前端安全

- [x] 麦克风权限按需请求
- [x] Token 存储在内存/localStorage（非 Cookie）
- [x] 用户输入进行转义处理
- [ ] 实现 CSP 头配置（待后续迭代）
- [ ] 实现 XSS 防护审计（待后续迭代）

## 隐私合规

- [x] 对话数据与用户绑定，不跨用户访问
- [x] 软删除支持数据恢复
- [ ] 实现数据导出功能（GDPR，待后续迭代）
- [ ] 实现账户删除功能（待后续迭代）
- [ ] 隐私政策文档（待后续迭代）

## 日志与监控

- [x] 结构化日志输出
- [x] 敏感数据不记录在日志中
- [ ] 实现安全事件审计日志（待后续迭代）
- [ ] 实现异常告警机制（待后续迭代）
