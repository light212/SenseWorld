## 一、设计概述

### 1.1 设计愿景

为 SenseWorld 后台管理系统打造**专业、高效、易用**的管理界面，让运维人员在 30 秒内完成核心操作。

### 1.2 设计原则

| 原则 | 描述 |
|------|------|
| **数据优先** | 后台核心是数据展示，图表、表格清晰易读 |
| **操作高效** | 高频操作一键可达，配置修改即时生效 |
| **状态可见** | 配置变更、系统状态、告警信息实时反馈 |
| **安全可控** | 敏感信息脱敏显示，关键操作二次确认 |

### 1.3 目标用户

| 用户角色 | 核心需求 | 高频操作 |
|----------|----------|----------|
| **运维人员** | 快速切换模型配置 | 配置修改、日志查询 |
| **产品经理** | 查看用量和费用数据 | 用量监控、趋势分析 |

## 七、交互设计

### 7.1 用户旅程

#### Journey: 模型配置修改

```
进入页面 → 查看当前配置 → 点击编辑 → 修改参数 → 点击保存 → 等待验证 → 看到成功提示
   │            │              │           │           │           │            │
   ↓            ↓              ↓           ↓           ↓           ↓            ↓
 加载配置    理解现状      打开表单    输入新值    确认操作    Spinner    Toast提示
                                                          │
                                              验证失败 ←─┘→ 成功保存
                                                  │
                                                  ↓
                                            显示错误，返回编辑
```

### 7.2 操作流程

#### 配置保存流程

```
用户点击"保存"
    │
    ▼
表单验证 ──失败──▶ 显示字段错误，返回编辑
    │ 成功
    ▼
显示确认弹窗 ──取消──▶ 返回编辑
    │ 确认
    ▼
按钮显示"保存中..."
    │
    ▼
API 调用 ──失败──▶ Toast: "保存失败" + 重试按钮
    │ 成功
    ▼
Toast: "配置已保存，5秒内生效"
    │
    ▼
配置热更新生效
```

#### 日志查询流程

```
用户进入日志页面
    │
    ▼
显示筛选器 + 骨架屏
    │
    ▼
用户输入筛选条件
    │
    ▼
防抖等待 300ms
    │
    ▼
显示加载动画
    │
    ▼
数据返回 ──无数据──▶ 空状态提示
    │ 有数据
    ▼
渲染表格（20条/页）
    │
    ▼
用户点击某行
    │
    ▼
展开详情抽屉
```

### 7.3 反馈机制

| 操作 | 成功反馈 | 失败反馈 |
|------|----------|----------|
| 保存配置 | 绿色 Toast "配置已保存，5秒内生效" | 红色 Toast "保存失败，请重试" |
| 删除配置 | 红色 Toast "已删除" | 红色 Toast "删除失败" |
| 刷新数据 | 数据更新动画 | 红色 Toast "刷新失败" |
| 导出日志 | 下载文件 + Toast "导出成功" | 红色 Toast "导出失败" |
